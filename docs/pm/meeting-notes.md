### 一、项目背景介绍 (PM Zhangjiasen)

**产品定位:**
- Notebook 是一款轻量级 Web 版 Markdown 笔记工具
- 核心理念：「专注写作，本地优先」
- 所有数据存储在用户本地文件系统，无需云端同步，保护用户隐私

**目标用户:**
1. 开发者 - 记录技术笔记、代码片段
2. 学生/研究者 - 整理学习笔记、文献摘要
3. 写作者 - 简洁的写作环境
4. 知识管理者 - 管理个人知识库

**项目周期:** 4 周 (2026-03-07 ~ 2026-04-03)

---

### 二、技术架构方案 (CTO Gilfoyle)

**技术栈决策:**

| 领域 | 选型 | 理由 |
|------|------|------|
| 前端框架 | React 18 + TypeScript 5 | 组件化开发，类型安全 |
| 构建工具 | Vite 5 | 快速开发启动，优秀 HMR |
| Markdown 编辑器 | Monaco Editor | VS Code 同款，功能强大 |
| 状态管理 | Zustand | 轻量级，比 Redux 简洁 |
| UI 组件库 | shadcn/ui + Tailwind CSS | 高度可定制，现代设计 |
| 后端框架 | Node.js 20 + Express 4 | 成熟稳定，与前端技术栈统一 |
| 数据库 | MySQL 8 (Docker) | 生产级数据库，支持多项目扩展 |
| 搜索 | FlexSearch | 轻量级，支持中文分词 |
| 部署 | Docker Compose | 环境一致性，简化部署 |

**重要技术决策:**

1. **MySQL vs SQLite:** 选择 MySQL 8
   - 支持多项目扩展
   - Docker 友好，易于部署管理
   - 生产级数据库，便于团队协作

2. **Docker 部署:** 
   - 所有数据卷映射到 `/home/ubuntu-user/docker` 目录
   - MySQL 数据：`/home/ubuntu-user/docker/notebook/mysql/data`
   - 应用数据：`/home/ubuntu-user/docker/notebook/app/data`

3. **本地文件存储:** 笔记内容仍存储为 Markdown 文件
   - 用户数据完全可控
   - 支持 Git 版本控制
   - MySQL 仅用于索引和元数据
