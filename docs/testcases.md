## 1. 笔记 CRUD 功能

### TC-001: 创建笔记
- **前置条件**: 应用正常运行
- **测试步骤**:
  1. 点击"+新建"按钮
  2. 输入标题"测试笔记"
  3. 输入内容"# 测试\n\n这是测试内容"
  4. 等待 500ms
- **期望结果**: 
  - 笔记创建成功
  - Network 中有 POST /api/notes 请求
  - 笔记出现在列表中
- **优先级**: P0

### TC-002: 读取笔记
- **前置条件**: 存在至少一篇笔记
- **测试步骤**:
  1. 刷新页面
  2. 点击笔记列表中的笔记
- **期望结果**:
  - 笔记列表正确显示
  - 右侧编辑器显示笔记内容
- **优先级**: P0

### TC-003: 更新笔记
- **前置条件**: 存在至少一篇笔记
- **测试步骤**:
  1. 选择一篇笔记
  2. 修改标题
  3. 修改内容
  4. 等待 500ms
- **期望结果**:
  - Network 中有 PUT /api/notes/:id 请求
  - 内容保存成功
  - 刷新后内容不丢失
- **优先级**: P0

### TC-004: 删除笔记
- **前置条件**: 存在至少一篇笔记
- **测试步骤**:
  1. 悬停在笔记上
  2. 点击删除按钮
  3. 确认删除
- **期望结果**:
  - Network 中有 DELETE /api/notes/:id 请求
  - 笔记从列表消失
  - 文件从文件系统删除
- **优先级**: P0

## 2. 自动保存功能

### TC-010: 自动保存触发
- **前置条件**: 已创建笔记
- **测试步骤**:
  1. 在编辑器中输入内容
  2. 停止输入，等待 500ms
  3. 观察 Network 标签
- **期望结果**:
  - 500ms 后触发 PUT 请求
  - 保存状态显示"已保存"
- **优先级**: P0

### TC-011: 防抖功能
- **前置条件**: 已创建笔记
- **测试步骤**:
  1. 快速连续输入多个字符
  2. 观察 Network 请求
- **期望结果**:
  - 多次输入只触发一次保存
  - 最后一次输入后 500ms 保存
- **优先级**: P1

### TC-012: 新建笔记立即编辑
- **前置条件**: 无
- **测试步骤**:
  1. 点击"新建"
  2. 立即开始输入内容（不等 API 返回）
  3. 观察是否保存成功
- **期望结果**:
  - 可以立即输入
  - 内容保存成功
  - 刷新后内容存在
- **优先级**: P0
